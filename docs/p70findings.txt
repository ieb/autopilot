  Key Findings

  1. The logs only captured standby mode — the pilot was never engaged, so no
  locked heading (65360), wind datum (65345), or button presses (65371) were
  recorded.
  2. No p70 keypad was on the bus during capture — PGN 65374 (heartbeat) and 65371
   (keypad messages) are absent. The logs only show EV-1 and ACU traffic.
  3. To control via p70, our firmware needs to:
    - Transmit PGN 65379 (pilot mode status) so the p70 shows the current mode
    - Transmit PGN 65359 (pilot heading) so the p70 shows current heading
    - Transmit PGN 65360 (locked heading) so the p70 shows target heading
    - Transmit PGN 65345 (wind datum) when in wind mode
    - Receive PGN 126720 keystroke commands (+1/-1/+10/-10/tack)
    - Receive PGN 126208 mode change commands (auto/standby/wind/track)
    - Receive PGN 65374 keypad heartbeat (detect p70 presence)
    - Transmit PGN 65288 (alarms) when faults occur
  4. Keystroke codes (in PGN 126720, 22-byte fast-packet):
    - 0x07F8 = +1°, 0x05FA = -1°
    - 0x08F7 = +10°, 0x06F9 = -10°
    - 0x22DD = Tack starboard, 0x21DE = Tack port
  5. Mode change (via PGN 126208 writing to PGN 65379):
    - Mode 0x00 SubMode 0x00 = Standby
    - Mode 0x40 SubMode 0x00 = Auto (compass)
    - Mode 0x00 SubMode 0x01 = Wind/Vane
    - Mode 0x80 SubMode 0x01 = Track
  6. Critical: Keypad heartbeat (PGN 65374) — the ACU expects a periodic heartbeat
   from the control head. Our firmware should either emulate this or the p70 will
  provide it since it's physically on the bus.

  Gaps to Fill

  To integrate with the p70, the firmware n2k.cpp needs new handlers for:
  - RX: PGN 126720 (keystrokes), PGN 126208 (mode commands), PGN 65374 (heartbeat
  detection)
  - TX: PGN 65379 (mode status), PGN 65359 (heading), PGN 65360 (locked heading),
  PGN 65345 (wind datum)

  The biggest unknown is the exact byte layout of PGN 126720 keystroke messages —
